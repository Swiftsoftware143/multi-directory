10. FAQ

How do I create a new directory?

From the Dashboard, click the "New Directory" button. Select a template (or start blank), give your directory a name and slug, then click Create. You'll be taken to the directory settings where you can customize categories, theme, and more.

Can I import listings from a spreadsheet?

Yes! Navigate to your directory and click "Import" in the listings section. Upload a CSV file with columns matching the listing fields (name, address, phone, etc.). The system will validate and import matching rows. You can download a sample CSV template from the import page.

How do I make my directory public?

Each directory has a status toggle in its settings. Set it to "Published" to make it accessible at its public URL (https://directory.swiftsoftware.net/d/your-directory-slug). Draft directories are only visible to admin users.

Is there a mobile app or mobile-responsive design?

The public directory pages are fully responsive and work great on mobile devices. There is no native mobile app, but the web interface works on all screen sizes. The admin dashboard is also responsive for on-the-go management.

Can I add custom fields to listings?

Absolutely. When editing a directory's template settings, you can add custom fields of various types: text, number, select dropdown, multi-select, date, URL, email, and file upload. These fields appear on both the add/edit listing forms and the public listing page.

How do I reset my password?

Go to Settings G�� Password. Enter your current password and new password. The new password must be at least 12 characters and include uppercase, lowercase, a number, and a special character. If you're locked out, contact the platform administrator.

Can I set up multiple administrators for my directory?

Yes. Under Settings G�� Team Members, click "Invite Member". Enter their email and select a role (Editor or Viewer). They'll receive an invitation email. Editors can manage listings and categories. Viewers can see the admin panel but cannot make changes.

What happens when I delete a directory?

Deleting a directory permanently removes all its listings, categories, theme settings, and analytics data. This action cannot be undone. The public URL will return a 404. You'll be asked to type the directory name to confirm deletion.

The Deals module lets you create limited-time offers, coupons, and promotions linked to directories and businesses. Think Groupon-style deals — discounted services, special offers, and coupon codes that appear on directory landing pages and business profiles.

🎿 Coupons & Deals

Create percentage-off or fixed-price deals. Set original price vs deal price. Coupon codes for in-store or online redemption.

⏰ Scheduled Offers

Set start and end dates. Deals auto-activate and auto-expire. Schedule promotions for holidays, slow seasons, or grand openings.

📊 Redemption Tracking

Each deal tracks how many times it's been claimed. Set redemption limits to control inventory. The "Claimed" count appears on each deal card.

Each deal is tied to a specific directory and business. Deals appear on business profile pages and directory landing pages.

Creating a Deal

  1. Click Deals in the sidebar navigation
  2. Click the + New Deal button in the top-right
  3. Fill in the modal form: Title, Description, Pricing (Original Price, Deal Price, Discount %, Currency)
  4. Select the Directory and enter the Business ID (UUID of the business)
  5. Set Status, Start/End dates, Redemption Limit, Coupon Code, and featured toggle
  6. Click Create Deal to save

Editing a Deal

Click any deal card in the Deals list to open the edit modal. Click Save Changes to persist updates.

Deleting a Deal

Click the Delete button (trash icon) on any deal card. Confirm the deletion in the dialog.

Tracking Redemptions

Each deal card displays a Claimed count showing how many times the deal has been redeemed. The Claim button lets you manually increment the redemption count for testing or manual fulfillment. The system also tracks the redemption limit and marks the deal as Sold Out when the limit is reached.

Deal Types

  • Deal — A standard discounted offer

12. Directory Structure & City Homepages

The Multi-Directory Platform uses a city-based URL structure for its public-facing directories. Each city (or location) gets its own dedicated subpage with unique SEO, and the root domain serves as a hub linking to all cities.

🏠 Root Domain = Homepage

directory.swiftsoftware.net shows a city grid homepage. Visitors can browse all available cities and click through to any directory.

🏛 Per-City Subpages

Each directory gets a dedicated page at /{city-slug} (e.g., /palm-bay, /orlando, /miami). These pages display businesses, deals, and content specific to that city.

🌎 Auto Geo-Redirect

When a visitor lands on the homepage, their IP is used to detect their approximate city. A soft banner offers to redirect them to their nearest city directory. Non-blocking and dismissible.

🔎 Per-City SEO

Each city directory has unique SEO settings: meta title, meta description, Open Graph tags, JSON-LD structured data (LocalBusiness schema), and its own sitemap entry.

URL Path Structure

Homepagehttps://directory.swiftsoftware.net/City grid with links to all directories
City Directoryhttps://directory.swiftsoftware.net/palm-bayBusiness listings, deals, categories for that city
Business Detailhttps://directory.swiftsoftware.net/palm-bay/business-slugFull business profile, deals, reviews
Admin Dashboardhttps://directory.swiftsoftware.net/#/...Hash-based SPA routing for admin panel

How City Directories Work

  1. Create a directory - Go to Admin → Directories → New Directory. Give it a name (e.g., "Palm Bay") and a slug (e.g., "palm-bay").
  2. City page auto-creates - The slug becomes a public URL at /{slug}. The city page inherits the directory template, theme, and categories.
  3. Add businesses - Add businesses to the directory as normal. They appear on the city page with full listings.
  4. Configure SEO - Under Admin → SEO, select the directory and set per-city meta title, description, and keywords. The sitemap auto-generates entries for each city.
  5. Homepage updates - The root homepage automatically lists all published directories as clickable city cards.

Geo-Redirect Setup

The geo-redirect is a soft, client-side check:

  • Visits a free IP geolocation API to detect the visitor’s city from their IP address
  • Matches the detected city to the nearest available directory slug
  • Shows a dismissible banner: "You appear to be near [City] → Visit [City] Directory"
  • The visitor clicks the link to navigate, or dismisses the banner to stay on the homepage
  • No forced redirects — always non-blocking and optional
